В SIMAI Framework 4 (SF4) страница собирается компонентом грида из **представления (view)** и набора **блоков**. Важно понимать два слоя:

* **view** — это сохранённая конфигурация “как собрать область” (строки/колонки/area-шаблоны \+ параметры). В актуальной схеме это не «просто шаблон», а файл, который хранит итоговый набор параметров, сформированный редактором.  
* **block** — единица вывода внутри area: у блока есть PHP-шаблон и (при необходимости) набор параметров, которые редактор может показывать и сохранять.

Практическое правило разработки: вы работаете через слой сайта `{site_dir}/simai.data` — добавляете/копируете/правите view и блоки там, а “платформенный” код и инфраструктуру ядра не трогаете напрямую.
