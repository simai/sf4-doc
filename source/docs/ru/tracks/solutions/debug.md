# Отладка и диагностика

## Назначение
Фиксирует подтверждённые точки входа для проверки и отладки решений на SF4.

## Где применяется
Административная панель SF4, файловая структура `{site_dir}/simai.data`, компоненты и мастер (wizard) из модуля `simai.framework`.

## Где находится
- Панель администратора с режимом правки и сбросом кеша: `source/sf4doc.md`.
- Каталоги данных и конфигов для проверки наличия файлов: `source/structure.json`, `source/sf4-structure.enriched.json`.

## Требования
Администраторские права для доступа к панели и к записи в `{site_dir}/simai.data`; установленные модули `simai.framework` и `simai.property`.

## Как это работает
Не найдено в локальных данных. Искомое: порядок логирования/отладки внутри компонентов SF4. Искали в `source/file-functional.md`, `source/file-frontend.md`, `source/sf4-structure.enriched.json`.

## Настройка и параметры
Не найдено в локальных данных. Искомое: настройки логирования/режимов debug для компонентов SF4. Искали в `source/structure.json`, `source/modules`.

## Примеры из репозитория
Источник: `source/sf4doc.md`
```
- **BITRIX:** Подготовлена панель администратора, со следующими возможностями:
- переход в режим администрирования;
- включение панели битрикса;
- режим правки;
- сброс кеша;
- режми редактирования данных;
- настроки сайта;
- настройки раздела;
- настройки страницы;
```

## Типичные ошибки и диагностика
- Нет прав на `{site_dir}/simai.data` → правки не применяются; проверить права и доступ к каталогу.
- Отсутствуют нужные view/block → сверить наличие файлов по структуре из `source/structure.json`.
- После правок нет изменений → включить режим правки и выполнить сброс кеша через панель.

## Что должен уметь читатель после
- Открыть административную панель SF4 и включить режим правки.
- Проверять наличие нужных файлов по структуре `{site_dir}/simai.data`.
- Сбрасывать кеш после изменений.
