| Code | Назначение | Реализация | Ключевые параметры | Примечания |
| :---- | :---- | ----: | :---- | :---- |
| `option.export.data` | Экспорт настроек модулей Bitrix | `action.php` | `parameter`: `module => [option1, option2, ...]` | Читает опции через `COption::GetOptionString`. |
| `option.import.data` | Импорт настроек модулей Bitrix | `action.php` | **Требует уточнения** (ожидается структура `module => option => value`) | В локальных данных есть исполняемый код, но схему входных данных лучше зафиксировать один раз в документации. |
| `usergroup.export.data` | Экспорт групп пользователей | `action.php` | **Требует уточнения** | Обычно нужен контракт: какие поля сохраняются (STRING\_ID/NAME/… ). |
| `usergroup.import.data` | Создание/обновление групп | `action.php` | **Требует уточнения** | Аналогично: по какому ключу матчим (STRING\_ID?) и что обновляем. |
| `shortlink.export.data` | Экспорт коротких ссылок | `action.php` | (как правило без параметров) | Делает выборку через `CBXShortUri::GetList()` и отдаёт массив. |
| `shortlink.import.data` | Импорт коротких ссылок | `action.php` | `parameter[file]` — файл-источник | Внутри берёт текущий site по входным данным (`site`), читает массив и добавляет через `CBXShortUri::Add`. |
